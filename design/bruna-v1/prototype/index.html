Perguntas frequentes

flowtype envia minha voz pra algum servidor? ▾
Por default, áudio é enviado pra Groq (cloud) pra transcrição rápida. Você pode ativar "Forçar fallback local" nas Settings — aí tudo roda offline via faster-whisper bundled, áudio nunca sai da sua máquina.
Groq é mesmo gratuito? Tem limite? ▾
Sim. Free tier do Groq dá 14.400 requests/dia por key. Como o flowtype suporta até 3 keys em pool, você pode chegar a 43.200 req/dia — mais que suficiente pra uso intenso. Quando estoura, cai automaticamente pro local.
Quais requisitos de hardware? ▾
Windows 10+, 2GB RAM livre. Pra fallback local funcionar bem, recomendado 4 cores e 4GB RAM (modelo small.en consome ~2GB enquanto roda).
Por que o Windows mostra "SmartScreen warning"? ▾
v0.1 não tem code signing (certificado EV custa $300+/ano). Clique "Mais informações" → "Executar mesmo assim". Vamos resolver isso em versão futura.
Quando sai pra Mac/Linux? ▾
v0.1 é Windows-first. Mac (com a nova permission model do System Events) e Linux (X11+Wayland separados) estão no roadmap pra v0.3.
Como funciona o vocabulário custom? ▾
Você cadastra pares "term_wrong → term_correct" nas Settings. Pós-transcrição, antes de colar, flowtype substitui. Útil pra nomes próprios (Kunha → Cunha), siglas técnicas, jargões específicos do seu trabalho.
Funciona offline? ▾
Sim. Sem internet, cai automaticamente pro faster-whisper local (modelo small.en bundled, ~140MB). Latência sobe pra ~3s mas funciona.
Onde está o código fonte? ▾
MIT license. Repositório em github.com/[…]/flowtype. PRs e issues bem-vindos.
Coleta algum telemetry? ▾
Não por default. Telemetria é opt-in (Settings → Privacidade) e mesmo ativada só registra contadores agregados (req count, latency p50), nunca conteúdo de transcrição.
Como contato suporte? ▾
Abra uma issue no GitHub (preferencial — outros podem se beneficiar) ou email support@flowtype.app.
